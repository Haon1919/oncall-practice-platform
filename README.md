# On-Call Practice Platform

Welcome to the On-Call Practice Platform! This application is designed to simulate real-world software engineering on-call scenarios. It leverages the power of Google's Gemini API to dynamically generate buggy applications, deploy them locally using Docker, and provide a realistic environment for you to practice your debugging and incident response skills.

## 🌟 Features

*   **Dynamic Scenario Generation:** Describe an application you want to debug. Gemini will build a functional Node.js/Express app, intentionally inject a subtle logical bug or configuration error, and deploy it locally.
*   **Simulated Support Inbox:** Receive realistic, slightly frustrated "support tickets" generated by AI from the perspective of a user experiencing the injected bug.
*   **Cloud Console Clone:** View live, streaming logs directly from the generated Docker container in a mock AWS, GCP, or Azure console interface.
*   **AI Senior Engineer Assistant:** Chat with an AI mentor who knows the secret bug. It won't give you the answer directly but will provide hints, ask guiding questions, and help you navigate the investigation process.
*   **Local Execution:** Everything runs locally on your machine using Docker, ensuring a safe and isolated practice environment.

## 🛠️ Prerequisites

Before you begin, ensure you have the following installed on your system:

1.  **Docker Desktop:** Required to run the main platform and to spin up the dynamically generated practice containers.
    *   [Mac Installation](https://docs.docker.com/desktop/install/mac-install/)
    *   [Windows Installation](https://docs.docker.com/desktop/install/windows-install/)
    *   [Linux Installation](https://docs.docker.com/desktop/install/linux-install/)
2.  **Google Gemini API Key:** You need an API key to power the scenario generation and the AI assistant.
    *   Get one from Google AI Studio.

## 🚀 Setup Instructions

1.  **Clone the repository** (if you haven't already):
    ```bash
    git clone <repository-url>
    cd oncall-practice-platform
    ```

2.  **Set up Environment Variables:**
    Copy the example environment file and add your Gemini API key.
    ```bash
    cp .env.example .env
    ```
    Open the `.env` file and replace `your_api_key_here` with your actual Gemini API key:
    ```env
    GEMINI_API_KEY=AIzaSyYourActualKeyHere...
    ```

3.  **Run the Setup Script:**
    We've included a script to verify that Docker is installed and running correctly on your system.
    ```bash
    ./setup.sh
    ```
    *If the script is not executable, run `chmod +x setup.sh` first.*

## 🏃‍♂️ Running the Application

The easiest way to run the platform is using Docker Compose. This ensures the application has the necessary access to your local Docker daemon to spawn the practice containers.

```bash
docker-compose up --build
```

Once the build is complete and the container is running, open your web browser and navigate to:

**http://localhost:3000**

## 🎮 How to Play (Workflow)

1.  **Dashboard:** Start by navigating to the Dashboard. Enter a name and description for an application (e.g., "User Authentication Service" that "handles login and registration"). Select your preferred simulated cloud provider and click "Generate Scenario".
2.  **Inbox:** Once the scenario is active, check your Inbox. You will receive a new support ticket describing the symptoms of the bug from a user's perspective.
3.  **Cloud Console:** Head over to the Cloud Console to view the live logs of the running application. Look for errors, stack traces, or unusual behavior.
4.  **AI Assistant:** If you get stuck, open the AI Assistant chat. Explain what you've found so far and ask for a hint. Remember, the assistant acts like a senior engineer and will guide you, not do the work for you.
5.  **Resolve:** Use the clues from the ticket, the logs, and the assistant to figure out what the injected bug is! (Note: Currently, fixing the bug involves understanding it; modifying the generated container's code live is an advanced exercise).

## 🏗️ Tech Stack

*   **Frontend:** Next.js 15 (App Router), React, Tailwind CSS, Zustand (State Management), Lucide React (Icons).
*   **Backend:** Next.js API Routes, Node.js.
*   **AI Integration:** `@google/genai` SDK (Gemini 2.5 Flash model).
*   **Infrastructure:** Docker, Dockerode (Docker Remote API for Node.js).

## ⚠️ Important Notes

*   **Docker Socket Mount:** The `docker-compose.yml` file mounts your host's Docker socket (`/var/run/docker.sock`) into the application container. This is necessary for the app to create and manage the practice containers. Ensure you understand the security implications of this in a production environment (though this app is intended for local practice only).
*   **Port Conflicts:** The generated practice applications will attempt to bind to random ports between 1024 and 65535. Ensure you don't have strict firewall rules blocking local port binding.
